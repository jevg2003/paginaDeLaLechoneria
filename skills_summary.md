# Resumen de Skills Instaladas en el Proyecto

Este archivo contiene un resumen en español de todas las skills (habilidades para el agente) descargadas en este proyecto, explicando brevemente qué hacen y cuándo es mejor utilizarlas.

---

## 🎨 UI/UX y Estilos

### 1. **web-design-guidelines**
*   **Qué hace:** Proporciona las directrices de diseño web moderno de Vercel (Mobile-First, espaciados, contraste y accesibilidad).
*   **Cuándo usarlo:** Cuando quieras que revise, audite o cree un diseño de página para asegurarte de que se vea premium, profesional y limpio.

### 2. **tailwind-4-docs**
*   **Qué hace:** Documentación y mejores prácticas para el uso de Tailwind CSS v4.
*   **Cuándo usarlo:** Cuando tengas dudas sobre nuevas directivas, utilidades o configuración de Tailwind v4 en el proyecto.

### 3. **tailwind-design-system**
*   **Qué hace:** Guías para construir un sistema de diseño escalable y consistente usando Tailwind.
*   **Cuándo usarlo:** Para definir tokens de diseño (colores, fuentes, bordes) globales y reutilizables en el proyecto.

### 4. **shadcn**
*   **Qué hace:** Guía de uso e instalación de componentes de shadcn/ui.
*   **Cuándo usarlo:** Para agregar nuevos componentes interactivos accesibles (modales, popovers, selectores, menús) estructurando bien el código.

### 5. **improve** (Shadcn Improve)
*   **Qué hace:** Mejores prácticas y refactorización para componentes de shadcn/ui.
*   **Cuándo usarlo:** Cuando quieras limpiar, simplificar o mejorar componentes de UI ya existentes en tu código.

## 🎬 Animaciones y 3D

### 6. **gsap-core**
*   **Qué hace:** Fundamentos y timelines del motor de animación GSAP.
*   **Cuándo usarlo:** Para crear movimientos y animaciones básicas o complejas de elementos del DOM (textos, botones, imágenes) de forma fluida.

### 7. **gsap-scrolltrigger**
*   **Qué hace:** Animaciones sincronizadas con el scroll del usuario mediante GSAP.
*   **Cuándo usarlo:** Para animar la aparición de secciones o cambios de estado visuales en la página a medida que el usuario navega verticalmente.

### 8. **threejs-animation**
*   **Qué hace:** Animaciones y renderizado 3D interactivo en la web usando Three.js y React Three Fiber.
*   **Cuándo usarlo:** Para programar comportamientos 3D complejos en scroll (por ejemplo, que un objeto 3D rote o interactúe).

### 9. **spline-interactive**
*   **Qué hace:** Integración de escenas 3D interactivas creadas con la herramienta de diseño Spline.
*   **Cuándo usarlo:** Cuando quieras insertar modelos 3D listos (como una galleta) y hacer que hagan transiciones o animaciones (como romperse) en base a eventos de scroll o clics sin codificar todo el WebGL a mano.

## 🗄️ Base de Datos y Supabase

### 10. **supabase**
*   **Qué hace:** Estándares para el uso del cliente JS de Supabase, autenticación de usuarios y queries.
*   **Cuándo usarlo:** Para resolver problemas de inicio de sesión, manejo de sesiones, tokens o consultas generales en Supabase.

### 11. **supabase-postgres-best-practices**
*   **Qué hace:** Optimización y diseño de esquemas en PostgreSQL.
*   **Cuándo usarlo:** Al crear nuevas tablas, optimizar índices, estructurar relaciones de base de datos o definir políticas de seguridad RLS.

---

## 🛠️ Refactorización y Calidad de Código

### 12. **request-refactor-plan**
*   **Qué hace:** Planeación paso a paso de refactorizaciones grandes de código.
*   **Cuándo usarlo:** Al planificar cambios importantes en la estructura de carpetas, dividir componentes gigantes o limpiar código redundante.

### 13. **review-and-refactor**
*   **Qué hace:** Pautas para revisar el código, reducir complejidad y simplificar archivos largos.
*   **Cuándo usarlo:** Cuando me pidas limpiar o dar más orden a archivos extensos de código redundante.

### 14. **find-skills**
*   **Qué hace:** El buscador del ecosistema de skills de agentes.
*   **Cuándo usarlo:** Para buscar e instalar nuevas herramientas o plugins cuando necesitemos una capacidad adicional.

## 🧭 Flujo de Trabajo y Colaboración (de gentle-ai)

### 15. **work-unit-commits**
*   **Qué hace:** Reglas para organizar commits como unidades de trabajo entregables (no por tipo de archivo), manteniendo tests y docs junto al código que verifican.
*   **Cuándo usarlo:** Antes de hacer commit de un cambio, para decidir qué agrupar y cómo escribir el mensaje.

### 16. **gentle-ai-chained-pr**
*   **Qué hace:** Estrategia para dividir cambios grandes (>400 líneas) en PRs encadenados o apilados, con diagrama de dependencias.
*   **Cuándo usarlo:** Cuando un cambio es demasiado grande para revisar de una vez.

### 17. **cognitive-doc-design**
*   **Qué hace:** Patrones para escribir documentación que reduce la carga cognitiva del lector (respuesta primero, chunking, tablas en vez de prosa larga).
*   **Cuándo usarlo:** Al escribir READMEs, guías, descripciones de PR o cualquier doc que deba entenderse rápido.

### 18. **comment-writer**
*   **Qué hace:** Reglas de tono para comentarios de PR/issues: directos, cálidos, cortos, sin relleno, en el idioma del contexto.
*   **Cuándo usarlo:** Al redactar feedback de revisión, respuestas a issues o mensajes async.
